# Clash of Prompt - AI Battle Arena

A turn-based battle RPG where your creativity is your weapon. Write prompts to attack enemies, and an AI Game Master narrates the battle, scores your creativity, and determines damage.

## Architecture

Split into two services:

```
clash-of-prompt/
├── src/                    Frontend (Vercel)
│   ├── app/api/            Thin proxies to backend
│   ├── components/         React UI components
│   └── lib/i18n.ts         Internationalization (EN/ID)
│
└── backend/                Backend (Fly.io / VPS)
    ├── server.ts           HTTP server
    ├── lib/                Game engine, Claude CLI, Gemini
    ├── Dockerfile
    └── fly.toml
```

- **Frontend**: Next.js 16 + React 19 + Tailwind CSS 4. Deployed on Vercel. API routes proxy requests to the backend.
- **Backend**: Node.js HTTP server with game logic. Uses Claude CLI (OAuth) for AI game master and Gemini API for battle image generation. Deployed on Fly.io or any Docker host.

## Features

- Turn-based RPG combat with AI-narrated battles
- Creativity scoring system (1-10) that scales damage
- Status effects: poison, stun, shield, heal, burn
- 3 enemies with unique personalities and weaknesses
- Comic-style battle images generated by Gemini
- Bilingual support (English / Bahasa Indonesia)

## Local Development

### Prerequisites

- Node.js 20+
- Docker (OrbStack recommended on macOS)
- Claude CLI authenticated (`claude auth login`)
- Gemini API key (optional, for battle images)

### Run locally

```bash
# Terminal 1: Backend (Docker)
cd backend
docker build -t clash-of-prompt-backend .
docker run -p 8080:8080 \
  -e CLAUDE_CODE_OAUTH_TOKEN=your-token \
  -e GEMINI_API_KEY=your-key \
  clash-of-prompt-backend

# Terminal 2: Frontend
npm install
npm run dev
```

Frontend runs at `http://localhost:3000`, backend at `http://localhost:8080`.

### Run backend without Docker

```bash
cd backend
npm install
npm run dev
```

Requires Claude CLI installed and authenticated locally.

## Deployment

### Backend (Fly.io)

```bash
cd backend
fly launch
fly secrets set CLAUDE_CODE_OAUTH_TOKEN=sk-ant-oat01-xxx
fly secrets set GEMINI_API_KEY=xxx
fly secrets set ALLOWED_ORIGINS=https://your-app.vercel.app
fly deploy
```

### Frontend (Vercel)

1. Push repo to GitHub
2. Import in Vercel
3. Set env var: `BACKEND_URL=https://your-app.fly.dev`
4. Deploy

## Environment Variables

### Frontend (.env.local)

| Variable | Required | Description |
|----------|----------|-------------|
| `BACKEND_URL` | Yes | Backend service URL |

### Backend

| Variable | Required | Description |
|----------|----------|-------------|
| `CLAUDE_CODE_OAUTH_TOKEN` | Yes | Claude CLI OAuth token |
| `GEMINI_API_KEY` | No | Gemini API key for battle images |
| `ALLOWED_ORIGINS` | No | CORS allowed origins (comma-separated) |
| `PORT` | No | Server port (default: 8080) |
